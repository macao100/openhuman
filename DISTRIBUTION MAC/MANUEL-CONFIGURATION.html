2. Modes de déploiement

Mode connecté (cloud backend)

# .env
OPENHUMAN_OFFLINE_MODE=false
BACKEND_URL=https://api.tinyhumans.ai

Nécessite un compte et un jeton JWT. Les appels LLM passent par le backend.

Mode hors-ligne (local uniquement)

# .env
OPENHUMAN_OFFLINE_MODE=true
OPENHUMAN_MODEL=llama3.2
OPENHUMAN_OLLAMA_BASE_URL=http://127.0.0.1:11434

Aucun backend cloud requis. Ollama doit être installé et un modèle téléchargé :

ollama pull llama3.2
ollama pull nomic-embed-text  # Embeddings pour la mémoire

Mode staging (développement)

# .env
OPENHUMAN_APP_ENV=staging
BACKEND_URL=https://staging-api.tinyhumans.ai

# app/.env.local
VITE_OPENHUMAN_APP_ENV=staging

3. Modèles IA

Ollama (local, recommandé)

# Télécharger un modèle
ollama pull llama3.2          # 3.8 Go — bon équilibre qualité/performance
ollama pull qwen2.5:7b        # 4.7 Go — alternative performante
ollama pull nomic-embed-text  # 274 Mo — nécessaire pour la mémoire sémantique

# Utiliser dans .env
OPENHUMAN_MODEL=llama3.2
OPENHUMAN_OLLAMA_BASE_URL=http://127.0.0.1:11434

LM Studio (local, interface graphique)

# .env
OPENHUMAN_LM_STUDIO_BASE_URL=http://127.0.0.1:1234/v1
OPENHUMAN_MODEL=# détecté automatiquement

BYOK (Bring Your Own Key)

# .env — exemple avec OpenAI
OPENHUMAN_MODEL=gpt-4o
# La clé API est gérée via l'interface utilisateur (Settings → AI)

4. Sécurité — Guardian N1-N3

NiveauLatenceRôle
N1 — Règles<1msBloque les patterns dangereux connus (paths, commandes)
N2 — Heuristique<10msDétecte exfiltration, entropie, payloads cachés
N3 — LLM<500msValide les plans ambigus via un modèle léger

Configurer l'accès agent dans config.toml ou via l'interface Settings → Agent access :

# ~/.openhuman/config.toml
[autonomy]
level = "supervised"    # readonly | supervised | full
workspace_only = true
allow_tool_install = false

5. Mémoire et stockage

DADOU stocke toutes les données localement dans ~/.openhuman/ :

~/.openhuman/
├── workspace/          # Espace de travail de l'agent
│   ├── memory_tree/    # Base SQLite de la mémoire
│   └── projects/       # Projets créés par l'agent
├── config.toml         # Configuration persistée
├── skills/             # Skills WASM installés
└── core.token          # Jeton d'authentification RPC
Attention : Ne pas supprimer ~/.openhuman/ sans avoir sauvegardé les données importantes. La mémoire, les préférences et l'historique y sont stockés.
