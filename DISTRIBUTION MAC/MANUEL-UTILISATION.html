📖 DADOU — Manuel d'utilisation

Exemples concrets d'utilisation de DADOU : desktop, CLI, JSON-RPC, et mode headless.

1. Lancement

Mode desktop — Interface graphique
# Depuis le répertoire du projet
cd ~/dadou
pnpm dev:app

# L'application s'ouvre dans une fenêtre native.
# Le core JSON-RPC démarre automatiquement en arrière-plan.
Mode serveur — JSON-RPC headless
cd ~/dadou
cargo run --release -- serve

[2026-06-18T10:00:00Z INFO] JSON-RPC server listening on http://127.0.0.1:7788/rpc
[2026-06-18T10:00:00Z INFO] Core token: ~/.openhuman/core.token

Le serveur JSON-RPC accepte les appels authentifiés. Utilisable avec n'importe quel client HTTP.

Mode CLI — Commande ponctuelle
# Aide générale
cargo run --release -- help

# Statut du système
cargo run --release -- status

# Health check
cargo run --release -- health
{ "status": "ok", "version": "0.56.0" }

2. Exemples — Mode hors-ligne (Ollama)

Prérequis : Ollama installé, modèle téléchargé (ollama pull llama3.2), OPENHUMAN_OFFLINE_MODE=true dans .env.
Conversation simple
# Via le desktop (pnpm dev:app) : l'interface de chat est disponible.
# Ou en CLI :
cargo run --release -- chat "Explique-moi le pattern Observer en Rust"
Analyse de code local
# Dans l'interface desktop : ouvrir le Chat, taper :
Analyse le fichier src/main.rs et propose des améliorations

# DADOU lit le fichier, l'envoie au modèle local, et répond.
# Toute l'analyse reste sur la machine — rien ne part sur le réseau.

3. Exemples — Gestion de fichiers

Créer un projet
# Demander à DADOU de créer un nouveau projet Rust
Crée un projet Rust "mon-api" avec Axum, un endpoint /health et des tests

DADOU crée la structure dans ~/OpenHuman/projects/mon-api/
├── Cargo.toml
├── src/
│   └── main.rs       ← serveur Axum avec /health
└── tests/
    └── health_test.rs
Modifier un fichier existant
Dans src/main.rs, remplace tous les unwrap() par du error handling
avec anyhow et thiserror

# DADOU lit le fichier, applique les changements, et crée
# une sauvegarde de rollback automatiquement.
Annuler une modification (rollback)
# Annuler la dernière modification
cargo run --release -- undo --last

# Restaurer un fichier à un état antérieur
cargo run --release -- undo --before "2026-06-18T09:00:00"

5. Exemples — JSON-RPC (intégration externe)

Appel RPC via curl
# Récupérer le token d'authentification
TOKEN=$(cat ~/.openhuman/core.token)

# Appeler une méthode RPC
curl -X POST http://127.0.0.1:7788/rpc \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "openhuman.health_snapshot",
    "params": {}
  }'

{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "status": "ok",
    "version": "0.56.0",
    "uptime_secs": 3600,
    "memory_mb": 128
  }
}
Script Python qui dialogue avec DADOU
#!/usr/bin/env python3
# client.py — Envoyer un message et lire la réponse
import requests, json, os

TOKEN = open(os.path.expanduser("~/.openhuman/core.token")).read().strip()
RPC = "http://127.0.0.1:7788/rpc"
HEADERS = {"Authorization": f"Bearer {TOKEN}", "Content-Type": "application/json"}

# Envoyer un message
resp = requests.post(RPC, json={
    "jsonrpc": "2.0", "id": 1,
    "method": "openhuman.channel_web_chat",
    "params": {"message": "Quelle est la capital du Burkina Faso ?"}
}, headers=HEADERS).json()

print(json.dumps(resp, indent=2, ensure_ascii=False))

6. Maintenance

Sauvegarde

# Sauvegarder tout le workspace DADOU
tar -czf dadou-backup-$(date +%Y%m%d).tar.gz ~/.openhuman

Mise à jour

cd ~/dadou
git pull origin master
pnpm install
cargo build --release --bin openhuman-core
pnpm build

Nettoyage

# Nettoyer les artefacts de build Rust (libère 2-10 Go)
cargo clean

# Nettoyer le cache pnpm
pnpm store prune
Astuce : La mémoire de DADOU s'améliore avec l'usage. Plus vous l'utilisez, plus il comprend vos préférences et votre contexte de travail.
