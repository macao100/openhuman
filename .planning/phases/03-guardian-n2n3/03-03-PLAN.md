    **Etape 4:** Implementation de `evaluate()`:

    ```rust
    pub async fn evaluate(&self, tool_name: &str, args: &serde_json::Value,
                          command: Option<&str>, file_path: Option<&str>) -> GuardianPipelineResult {
        // Stage 1: N1
        let n1_result = self.n1.evaluate(tool_name, args, command, file_path).await;

        if !n1_result.allowed {
            return GuardianPipelineResult {
                allowed: false,
                blocked_by: "n1".into(),
                n1: n1_result,
                n2: None,
                n3: None,
            };
        }

        // Stage 2: N2 (synchrone, <10ms)
        let n2_result = self.n2.evaluate(tool_name, args, command, file_path);

        if !n2_result.allowed {
            return GuardianPipelineResult {
                allowed: false,
                blocked_by: "n2".into(),
                n1: n1_result,
                n2: Some(n2_result),
                n3: None,
            };
        }

        // Stage 3: N3 (uniquement si N2 escalade)
        let n3_result = if n2_result.escalate && self.n3.config.enabled {
            let result = self.n3.evaluate(tool_name, args, command, file_path, &n2_result.scores).await;
            Some(result)
        } else {
            None
        };

        // Decision finale
        let n3_blocks = n3_result.as_ref().map_or(false, |r| r.should_block());

        GuardianPipelineResult {
            allowed: !n3_blocks,
            blocked_by: if n3_blocks { "n3" } else { "none" },
            n1: n1_result,
            n2: Some(n2_result),
            n3: n3_result,
        }
    }
    ```

    ```rust
    /// Construit le message de blocage a partir du resultat du pipeline.
    fn build_block_reason(result: &GuardianPipelineResult) -> String {
        match result.blocked_by.as_str() {
            "n1" => {
                let blocks: Vec<_> = result.n1.rule_results.iter()
                    .filter(|r| r.action == RuleAction::Block)
                    .map(|r| format!("[{}: {}]", r.rule_name, r.reason))
                    .collect();
                format!("N1 rules: {}", blocks.join("; "))
            }
            "n2" => {
                if let Some(ref n2) = result.n2 {
                    let blocks: Vec<_> = n2.scores.iter()
                        .filter(|s| s.score >= 0.7)  // BLOCK_THRESHOLD
                        .map(|s| format!("[{}: {} (score={})]", s.triggered_by, s.reason, s.score))
                        .collect();
                    format!("N2 detection: {}", blocks.join("; "))
                } else {
                    "N2 blocked (unknown)".into()
                }
            }
            "n3" => {
                if let Some(ref n3) = result.n3 {
                    format!("N3 validation: {} (verdict={:?})", n3.reason, n3.verdict)
                } else {
                    "N3 blocked (unknown)".into()
                }
            }
            _ => "unknown".into(),
        }
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| N1 result -> N2 decision | Si N1 est bypass, N2 ne rattrape pas (intentionnel: N1 est le niveau de confiance principal) |
| N2 decision -> N3 call | N3 n'est appele que si N2 est incertain. Si N2 est mal configure, N3 peut etre contourne. |
| Pipeline result -> tool execution | Le resultat du pipeline controle l'execution. Integrite du resultat critique. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | Pipeline decision flow | mitigate | Early exit apres N1/N2. Pas de bypass possible (le pipeline est la seule route vers tool.execute()). |
| T-03-10 | Elevation of Privilege | N3 disabled config | mitigate | Si N3 est disabled et N2 escalade, le pipeline bloque l'action (fail-closed, D-42). |
| T-03-11 | Information Disclosure | N3Result event | accept | Contient le verdict et la raison du LLM. Pas de donnees utilisateur. |
| T-03-12 | Denial of Service | Pipeline latency | mitigate | Early exit rapide (N2 pas appele si N1 bloque, N3 pas appele si N2 bloque). N3 timeout 450ms. |
</threat_model>
