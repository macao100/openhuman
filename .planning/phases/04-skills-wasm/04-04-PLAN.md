<objective>
Build a static analysis engine that scans third-party skill source code for suspicious patterns (dangerous imports, unauthorized filesystem writes, network calls) before activation.

Purpose: Prevent malicious or poorly-written skills from executing dangerous operations inside the DADOU sandbox. Static analysis is a cheap pre-check that runs before any WASM code is loaded into the wasmtime runtime.

Output:
- `src/openhuman/skills/static_analysis.rs` — Source code scanner with rule definitions, scorer, and result types
- Tests demonstrating detection of suspicious imports, writes, and network calls
</objective>

       ```rust
       #[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
       pub enum AnalysisVerdict {
           Pass,
           Warn,   // Ambiguous patterns found, review recommended
           Block,  // Dangerous patterns found, activation prevented
       }
       ```

    2. **`AnalysisResult` struct**:
       ```rust
       pub struct AnalysisResult {
           pub verdict: AnalysisVerdict,
           pub findings: Vec<AnalysisFinding>,
       }
       
       pub struct AnalysisFinding {
           pub severity: FindingSeverity,
           pub file: String,
           pub line: usize,
           pub pattern: String,     // Which rule matched
           pub snippet: String,     // The matching source line
       }
       
       pub enum FindingSeverity {
           Critical,  // Certain exploit: eval, subprocess
           High,      // Likely dangerous: os.system, socket
           Medium,    // Potentially dangerous: file write outside data dir
       }
       ```

    3. **AnalysisRule** struct — defines scan rules:
       ```rust
       pub struct AnalysisRule {
           pub name: &'static str,
           pub pattern: Regex,
           pub severity: FindingSeverity,
           pub description: &'static str,
       }
       ```

    4. **Default rules** (per D-48):
       ```rust
       pub fn default_rules() -> Vec<AnalysisRule> {
           vec![
               // Critical: Code execution primitives
               AnalysisRule::critical(r"eval\s*\(", "Use of eval()"),
               AnalysisRule::critical(r"exec\s*\(", "Use of exec()"),
               AnalysisRule::critical(r"std::process::Command", "Process execution"),
               AnalysisRule::critical(r"os\.system\s*\(", "Shell command execution"),
               AnalysisRule::critical(r"subprocess\..*\(call|run|Popen|check_output)", "Subprocess execution"),
               
               // High: Network access
               AnalysisRule::high(r"import\s+socket", "Socket import"),
               AnalysisRule::high(r"requests\.(get|post|put|delete)\s*\(", "HTTP request"),
               AnalysisRule::high(r"TcpStream::connect", "TCP connection"),
               AnalysisRule::high(r"\bcurl\s", "curl command"),
               AnalysisRule::high(r"\bwget\s", "wget command"),
               AnalysisRule::high(r"http::", "HTTP library usage"),
               
               // High: Filesystem outside allowed paths
               AnalysisRule::high(r"std::fs::(write|create_dir|remove|rename)\s*\(", "Filesystem write"),
               AnalysisRule::high(r"open\(.*['\"]/[^d]", "File open outside data dir"),
               
               // Medium: Information gathering
               AnalysisRule::medium(r"os\.environ", "Environment access"),
               AnalysisRule::medium(r"\$HOME|~\/|%USERPROFILE%", "Home directory reference"),
           ]
       }
       ```
       
       Helper constructors:
       ```rust
       impl AnalysisRule {
           fn critical(pattern: &str, desc: &str) -> Self {
               Self { name: desc, pattern: Regex::new(pattern).unwrap(), severity: FindingSeverity::Critical, description: desc }
           }
           fn high(pattern: &str, desc: &str) -> Self { ... }
           fn medium(pattern: &str, desc: &str) -> Self { ... }
       }
       ```

    5. **`scan_file(content: &str, file_path: &Path, rules: &[AnalysisRule]) -> Vec<AnalysisFinding>`**:
       - Iterate rules, scan each line
       - Return all matching findings with file path, line number, snippet

    6. **`scan_skill(skill_dir: &Path, permissions: &Permissions) -> Result<AnalysisResult>`**:
       - Walk `skill_dir/src/` recursively for source files (extensions: .rs, .py, .js, .ts, .sh)
       - For each file: read content, scan with rules, collect findings
       - Verdict logic:
         - Any Critical finding → Block
         - Any High finding AND not permitted → Block (for network: check permissions.network; for filesystem: check permissions.filesystem.write)
         - Any High finding BUT permitted → Warn (review recommended)
         - Any Medium finding → Warn
         - No findings → Pass
       - Filesystem permission check: a file write finding is downgraded from Block to Warn if the write path matches a pattern in `permissions.filesystem.write`

    7. **Helper `scan_file_for_writes(file_path: &Path) -> Vec<String>`** — extract filesystem write targets from source to validate against permissions

    **Conventions**:
    - Use `walkdir` crate (already in Cargo.toml) for directory recursion
    - Use `tracing::debug!("[static-analysis] scanning {file}: {n} findings")`
    - No unwrap() in production code
    - Rule patterns are case-insensitive by default
    - Files are read with `std::fs::read_to_string`, skip binary files
