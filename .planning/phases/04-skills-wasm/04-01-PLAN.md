  <files>

  <action>
    Create `src/openhuman/skills/manifest.rs` with:

    1. **`SkillManifest` struct** — top-level representation of `dadou-skill.yaml`:
       ```rust
       pub struct SkillManifest {
           pub name: String,                    // Required, trimmed, max 64 chars
           pub version: String,                 // Required, semver string
           pub description: Option<String>,     // Optional
           pub author: Option<String>,          // Optional author name
           pub gpg: Option<GpgConfig>,          // Optional GPG configuration
           pub permissions: Permissions,         // Default: deny-all
           pub wasm: WasmConfig,                 // Required: path + entry point
           pub dependencies: Vec<Dependency>,    // Optional, empty by default
       }
       ```

    2. **Sub-types**:
       - `GpgConfig { fingerprint: String }` — expected GPG key fingerprint
       - `WasmConfig { path: PathBuf, entry: String }` — path to WASM binary (relative to repo root), entry function name
       - `Permissions { filesystem: FilesystemPerms, network: bool }` — `FilesystemPerms { read: Vec<String>, write: Vec<String> }` (glob patterns for allowed paths)
       - `Dependency { name: String, version: String }`

    3. **`parse_manifest(yaml_str: &str) -> Result<SkillManifest, ManifestError>`** — validate and parse:
       - Reject empty `name`, `version`
       - `name` must match `^[a-zA-Z0-9_-]+$` (no spaces, safe for filesystem)
       - `version` must be non-empty
       - `wasm.path` must be present and not contain `..`
       - `network` default: `false` (deny by default)
       - Return `ManifestError` enum with `InvalidField(String)` variant

    4. **`ManifestError` enum** with `thiserror`:
       - `ParseError(String)` — YAML parsing failure
       - `InvalidField(String)` — validation failure
       - `MissingField(&'static str)` — required field absent

    5. **`default_permissions()`** — returns Permissions with all access denied:
       ```rust
       impl Default for Permissions {
           fn default() -> Self {
               Self { filesystem: FilesystemPerms { read: vec![], write: vec![] }, network: false }
           }
       }
       ```

    **Conventions** (per D-44):
    - Follow existing `src/openhuman/skills/ops_types.rs` naming/style
    - All structs derive `Debug, Clone, Serialize, Deserialize`
    - `#[serde(deny_unknown_fields)]` on manifest struct to reject unknown YAML keys
    - Use `anyhow::Result` for public API, `ManifestError` for typed errors internally
    
    **Tests** (`manifest_tests.rs`, `#[cfg(test)] mod tests` inline):
    - `parses_valid_manifest()` — full YAML with all fields
    - `rejects_missing_name()` — empty name = error
    - `rejects_missing_version()` — empty version = error
    - `rejects_path_traversal()` — `wasm.path` containing `..`
    - `rejects_unknown_yaml_keys()` — extra keys produce error
    - `default_permissions_deny_network()` — network = false by default
    - `default_permissions_empty_filesystem()` — read/write lists empty by default
    - `serde_roundtrip()` — struct -> YAML -> struct preserves data
  </action>

<task type="auto">
  <name>Task 2: Create TOML-based SkillsStore for installed skills tracking</name>
  <files>
    src/openhuman/skills/store.rs
    src/openhuman/skills/store_tests.rs
  </files>
  <action>
    Create `src/openhuman/skills/store.rs` with:

    1. **`InstalledSkill` struct** — per-skill entry in the store:
       ```rust
       pub struct InstalledSkill {
           pub name: String,
           pub version: String,
           pub commit_hash: String,
           pub enabled: bool,
           pub gpg_fingerprint: Option<String>,
           pub installed_at: String,         // ISO 8601
           pub last_audit_at: Option<String>, // ISO 8601
           pub audit_result: Option<String>,  // "pass" | "fail" | null
       }
       ```

    2. **`SkillsStore` struct**:
       ```rust
       pub struct SkillsStore {
           path: PathBuf,
           skills: HashMap<String, InstalledSkill>,
       }
       ```
       Methods:
       - `load() -> Result<Self>` — reads from `~/.openhuman/skills/store.toml`, returns empty store if file missing
       - `save(&self) -> Result<()>` — writes store to TOML file atomically (write to .tmp then rename)
       - `get(&self, name: &str) -> Option<&InstalledSkill>`
       - `get_mut(&mut self, name: &str) -> Option<&mut InstalledSkill>`
       - `list(&self) -> Vec<&InstalledSkill>` — all skills sorted by name
       - `upsert(&mut self, skill: InstalledSkill) -> Result<()>` — insert or update, then save
       - `remove(&mut self, name: &str) -> Result<bool>` — remove by name, then save. Returns true if existed
       - `set_enabled(&mut self, name: &str, enabled: bool) -> Result<()>` — toggle enabled state

    3. **Store file format** (`~/.openhuman/skills/store.toml`):
       ```toml
       [skills.skill-name]
       name = "skill-name"
       version = "0.1.0"
       commit_hash = "abc123def456..."
       enabled = true
       gpg_fingerprint = "A1B2C3D4..."
       installed_at = "2026-06-05T10:00:00Z"
       last_audit_at = "2026-06-05T10:05:00Z"
       audit_result = "pass"
       ```

    4. **Path resolution**: Base path = `~/.openhuman/skills/`. Create dirs if missing. Use `dirs::home_dir()`.
       - Store path: `~/.openhuman/skills/store.toml`
       - Skill data dirs: `~/.openhuman/skills/<name>/` (each skill gets its own subdirectory)

    5. **Atomic write pattern** (per D-45):
       ```rust
       fn atomic_write(path: &Path, content: &str) -> Result<()> {
           let tmp = path.with_extension("toml.tmp");
           std::fs::write(&tmp, content)?;
           std::fs::rename(&tmp, path)?;
           Ok(())
       }
       ```

    **Conventions**:
    - `to` and `from` are crate-level structs, not `pub use` re-exports from sub-modules
    - Use `toml::to_string_pretty` for human-readable output
    - No `unwrap()` in production code — use `?` with `anyhow::Context`
    - Log writes with `tracing::debug!("[skill-store] saved {} skills", n)`
    
    **Tests** (`store_tests.rs`, `#[cfg(test)] mod tests` inline):
    - `load_returns_empty_when_store_missing()` — file doesn't exist = empty store
    - `upsert_adds_new_skill()` — new skill appears in list
    - `upsert_updates_existing_skill()` — same name updates fields
    - `remove_deletes_skill()` — skill no longer in list after remove
    - `set_enabled_toggles_state()` — enabled flips from true to false
    - `save_and_load_roundtrip()` — save then load preserves all fields
    - `atomic_write_creates_and_renames()` — tmp file pattern works
    - `empty_store_sorted_list()` — list on empty store returns empty vec
    - Test on temp directories (use `tempfile::tempdir()`): `SKILLS_STORE_TEST_DIR` env var or tempfile
  </action>
