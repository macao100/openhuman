<objective>
Wrap all external data entering the LLM prompt in `<external_data source="..." trusted="false">` tags and teach the LLM to treat tagged content as data, not instructions.

Purpose: Prevent prompt injection through untrusted content (memory recall, skill outputs, web fetches, file reads). The LLM must see a clear visual and semantic boundary between system instructions and untrusted data.

Output:
- Migrated `<external_data>` tag format replacing `<untrusted-source>`
- System prompt `AntiInjectionSection` explaining the tag contract
- Extended wrapping to all external data entry points (not just memory)
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-03 | Tampering | tool_loop.rs wrapping | mitigate | Apply wrapping at the tool result level (closest to execution), not at prompt assembly — prevents bypass |
| T-05-04 | Information Disclosure | tool_loop.rs | accept | Wrapped content still reaches the LLM; tag does not redact content, only labels it |
